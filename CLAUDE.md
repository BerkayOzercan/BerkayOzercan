## index.html (Portfolio Site) Requirements

This is a single-file `index.html` — all CSS and JS must be inline (no external files except fonts and CDN libraries).

### Design Direction
- **Aesthetic:** Dark, editorial, slightly cinematic — draws from both game dev culture and modern design portfolios
- **NOT:** Generic blue/purple gradient corporate template
- **Fonts:** Use Google Fonts — pick something with character (e.g., a strong display font paired with a clean body font). Avoid Inter, Roboto, Arial.
- **Color palette:** Deep dark background (near-black), warm accent (amber, rust, or gold — fits "Regal Rust" branding), clean white text
- **Layout:** Full-width sections, large typography, grid-based project cards
- **Animations:** Subtle scroll reveal, hover states on project cards, smooth transitions
- **Navigation:** Sticky top nav with smooth scroll

### Site Sections (in order):
1. **Hero** — Name, title (multi-role), availability badge, CTA buttons (View Work / Contact)
2. **About** — Short bio, stat counters (games shipped, years exp, platforms)
3. **Skills** — Grouped by category: Game Dev / Web & Backend / Design & Creative / Tools & AI
4. **Projects** — Cards with image, title, description, tech tags, and link (if applicable). Show "In Progress" badge for active projects.
5. **Contact** — Email, phone, LinkedIn, GitHub, X
